# Ansible Roles & Playbooks — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement all Ansible roles and playbooks needed to bootstrap Slurm clusters, onboard and activate nodes, and propagate config changes — ready to be invoked by Aura Web (bootstrap) and Aura Agent (all subsequent operations).

**Architecture:** Single-responsibility roles tested with Molecule + Docker. Playbooks compose roles per operation. All cluster-specific values (node expressions, IPs, partitions) arrive as runtime vars in a JSON file generated by Aura Web/Agent. The `/mgmt` NFS share is the distribution point for configs and the munge key across all nodes.

**Tech Stack:** Ansible 8+ (ansible-core 2.15+), Molecule 6+ with Docker driver (`geerlingguy/docker-ubuntu2204-ansible`), Ubuntu 22.04 LTS target nodes, MySQL 8.0, community.mysql + community.general Ansible collections.

## Vars Contract

All playbooks accept a runtime vars file passed via `ansible-playbook -e @cluster-config.json`. This is the interface between Ansible and Aura Web/Agent. **Both Plan 2 (Agent) and Plan 3 (Web) must conform to this schema.**

```json
{
  "slurm_cluster_name": "sci-cluster-01",
  "slurm_controller_host": "slm-master",
  "mgmt_nfs_server": "192.168.1.1",
  "data_nfs_server": "192.168.1.1",
  "mgmt_nfs_path": "/mgmt",
  "data_nfs_path": "/aura-usrdata",
  "nfs_allowed_network": "192.168.1.0/24",
  "freeipa_server": "ipa.scicom.internal",
  "freeipa_domain": "scicom.internal",
  "aura_agent_nats_url": "nats://aura-web.scicom.internal:4222",
  "aura_agent_binary_src": "/tmp/aura-agent",
  "slurm_nodes": [
    {
      "expression": "slm-node-[01-10]",
      "cpus": 32,
      "gpus": 4,
      "memory_mb": 262144
    },
    {
      "expression": "slm-node-gpu-01",
      "cpus": 64,
      "gpus": 8,
      "memory_mb": 524288
    }
  ],
  "slurm_partitions": [
    {
      "name": "gpu",
      "nodes": ["slm-node-[01-10]", "slm-node-gpu-01"],
      "max_time": "24:00:00",
      "default": true
    }
  ],
  "slurm_hosts_entries": [
    {"hostname": "slm-master", "ip": "192.168.1.1"},
    {"hostname": "slm-node-01", "ip": "192.168.1.10"},
    {"hostname": "slm-node-02", "ip": "192.168.1.11"}
  ]
}
```

> `slurm_hosts_entries` is pre-expanded by Aura Web from bracket notation before passing to Ansible.
> `target_node` is passed additionally for onboard/activate operations: `-e target_node=slm-node-03`
> `user_uid` and `user_gid` are passed for user homedir operations: `-e user_uid=10001 -e user_gid=10001`

```yaml
    path: /etc/munge/munge.key
    owner: "{{ munge_user }}"
    group: "{{ munge_group }}"
    mode: '0400'

- name: Copy munge key to mgmt FS
  ansible.builtin.copy:

    dest: "{{ mgmt_nfs_path }}/munge.key"

    group: "{{ munge_group }}"
    mode: '0400'
    remote_src: true
  when:

- name: Copy munge key from mgmt FS on workers

    src: "{{ mgmt_nfs_path }}/munge.key"
    dest: /etc/munge/munge.key
    owner: "{{ munge_user }}"
    group: "{{ munge_group }}"
```

## Task 7: `nfs_client` Role

Mounts `/mgmt` and `/aura-usrdata` on worker nodes.

**Files:**
- Create: `ansible/roles/nfs_client/tasks/main.yml`

- [ ] **Step 1: Write `ansible/roles/nfs_client/tasks/main.yml`**

```yaml
---
- name: Install NFS client
  ansible.builtin.apt:
    name: nfs-common
    state: present
    update_cache: true

- name: Create mgmt mount point
  ansible.builtin.file:
    path: "{{ mgmt_nfs_path }}"
    state: directory
    owner: root
    group: root
    mode: '0755'

- name: Create data mount point
  ansible.builtin.file:
    path: "{{ data_nfs_path }}"
    state: directory
    owner: root
    group: root
    mode: '0755'

- name: Mount mgmt FS
  ansible.posix.mount:
    path: "{{ mgmt_nfs_path }}"
    src: "{{ mgmt_nfs_server }}:{{ mgmt_nfs_path }}"
    fstype: nfs
    opts: "rw,sync,hard,intr"
    state: mounted

- name: Mount data FS
  ansible.posix.mount:
    path: "{{ data_nfs_path }}"
```

Adds a node outside the originally declared range. Must update hosts on all existing nodes first.

```yaml
---
# Add a new node outside the originally declared range.
# Usage:
#   ansible-playbook -i inventory/hosts.ini \
#     -e @cluster-config.json \
#     -e target_node=slm-node-11 \
#     add_node.yml
#
# Required extra vars: target_node
# cluster-config.json must include the new node in slurm_nodes,

- name: Update hosts block on all existing cluster nodes
  hosts: slurm
  become: true
  roles:
    - hosts_block

- name: Onboard new worker node
  hosts: "{{ target_node }}"
  become: true
  roles:
    - common
    - chrony
    - nfs_client
    - munge
    - sssd
    - slurm_worker

- name: Update slurm.conf on controller and restart
  hosts: slurm_controllers
  become: true
  tasks:
    - name: Write updated slurm.conf
      ansible.builtin.template:
        src: roles/slurm_controller/templates/slurm.conf.j2
        dest: "{{ slurm_config_dir }}/slurm.conf"
        owner: "{{ slurm_user }}"
        group: "{{ slurm_group }}"
        mode: '0644'

    - name: Copy updated slurm.conf to mgmt FS
      ansible.builtin.copy:
        src: "{{ slurm_config_dir }}/slurm.conf"
        dest: "{{ mgmt_nfs_path }}/slurm.conf"
        remote_src: true
        owner: "{{ slurm_user }}"
        group: "{{ slurm_group }}"
        mode: '0644'

    - name: Restart slurmctld to pick up new node
      ansible.builtin.service:
        name: slurmctld
        state: restarted

  post_tasks:
    - name: Confirm new node visible in sinfo
      ansible.builtin.command: sinfo -N -n {{ target_node }} --noheader --format="%T"
      register: node_state
      changed_when: false
      retries: 6
      delay: 5
      until: node_state.stdout | trim in ['idle', 'idle~', 'down', 'allocated', 'future']
      failed_when: node_state.stdout | trim == ''

    - name: Print node state
      ansible.builtin.debug:
        msg: "Node {{ target_node }} state: {{ node_state.stdout | trim }}"
```

## Task 17: `propagate_config.yml` and `user_homedir.yml` Playbooks

**Files:**
- Create: `ansible/propagate_config.yml`
- Create: `ansible/user_homedir.yml`

- [ ] **Step 1: Write `ansible/propagate_config.yml`**

Copies configs from `/mgmt` to `/etc/slurm` on all nodes and restarts services. The Agent writes updated configs to `/mgmt` before invoking this playbook.

```yaml
---
# Propagate updated slurm configs from /mgmt to all cluster nodes.
# Usage:
#   ansible-playbook -i inventory/hosts.ini -e @cluster-config.json propagate_config.yml
#
# Pre-condition: Agent has already written updated configs to {{ mgmt_nfs_path }}/

- name: Propagate slurm config to all nodes
  hosts: slurm
  become: true
  tasks:
    - name: Copy slurm.conf from mgmt FS
      ansible.builtin.copy:
        src: "{{ mgmt_nfs_path }}/slurm.conf"
        dest: "{{ slurm_config_dir }}/slurm.conf"
        owner: "{{ slurm_user }}"
        group: "{{ slurm_group }}"
        mode: '0644'
        remote_src: true
      notify:
        - restart slurmctld
        - restart slurmd

    - name: Copy cgroup.conf from mgmt FS
      ansible.builtin.copy:
        src: "{{ mgmt_nfs_path }}/cgroup.conf"
        dest: "{{ slurm_config_dir }}/cgroup.conf"
        owner: "{{ slurm_user }}"
        group: "{{ slurm_group }}"
        mode: '0644'
        remote_src: true
      notify:
        - restart slurmctld
        - restart slurmd

  handlers:
    - name: restart slurmctld
      ansible.builtin.service:
        name: slurmctld
        state: restarted
      when: inventory_hostname in groups['slurm_controllers']

    - name: restart slurmd
      ansible.builtin.service:
        name: slurmd
        state: restarted
      when: inventory_hostname in groups['slurm_workers']
```
